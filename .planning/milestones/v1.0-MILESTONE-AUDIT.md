## Critical Gaps

### 🔴 RES-01: Graceful Shutdown Hangs (Unsatisfied)

**Issue:** `shutdown_event` is set on SIGINT/SIGTERM, but `KalshiIngestor` and `PolymarketIngestor` do not receive it. Their `BaseIngestor.run()` infinite `while True` loop only exits on `CancelledError` or exception, causing `asyncio.gather()` in `_main()` to hang indefinitely.

**Impact:** The "graceful" shutdown is actually a hang. The process must be killed with `SIGKILL`. Pending state may not be fully persisted.

**Fix:** Pass `shutdown_event` to ingestors and add shutdown checks in all infinite loops, or use `asyncio.wait_for` with `shutdown_event.wait()` on queue gets.

### 🔴 RES-03: Circuit Breaker Never Fires (Unsatisfied)

**Issue:** `RiskChecker.check()` is a **synchronous** method, but calls `self._bus.publish(...)` without `await`. Since `EventBus.publish()` is `async def`, this creates a coroutine object that is immediately garbage-collected. The `StrategyHaltedEvent` is **never delivered**.

**Impact:** When daily loss limit is breached repeatedly, the circuit breaker logs the trigger but does NOT actually halt the strategy via the event bus. The strategy continues running until the next `StrategyEngine` reload interval (up to 30s).

**Fix:** Change `RiskChecker.check()` to `async def check(...)` and `await self._bus.publish(...)`; update all call sites (`ExecutionEngine._handle_intent()`) to `await self._risk.check(...)`.

---

## Partial Gaps

### 🟡 RES-02: Health Check Incomplete

**Issue:** `cmd_health()` initializes `health["adapters"] = {}` but never populates it. It does not check whether adapters can reach exchanges, whether ingestors are connected, or whether engine tasks are alive.

**Impact:** A user running `python -m albert health` sees DB/strategy/position stats but cannot tell if the trading pipeline is actually alive.

### 🟡 MomentumV1 Redundant Kelly Computation

**Issue:** `MomentumV1.on_market_data()` calls `kelly_size()` and stores result in `size`, then ignores `size` and emits a bare `OrderIntent`. `ExecutionEngine._handle_intent()` recomputes Kelly sizing from scratch.

**Impact:** Wasted CPU cycles, potential confusion.
