<objective>
Fix two critical resilience bugs identified in the v1.0 milestone audit:
1. Graceful shutdown hangs because `asyncio.gather()` blocks forever on tasks that never exit
2. `RiskChecker.check()` calls async `EventBus.publish()` without await, so `StrategyHaltedEvent` is never delivered

Purpose: Close the two most severe gaps where the system cannot shut down cleanly and circuit breaker events are silently dropped.
Output: Modified source files with shutdown propagation and async risk checking.
</objective>

  <action>
Fix the shutdown anti-pattern where `asyncio.gather()` blocks forever because tasks are stuck on `queue.get()`, `websockets.connect()`, or `asyncio.sleep()`.

1. **albert/__main__.py** — Replace `await asyncio.gather(...)` with explicit task creation and cancellation:
   - Create each service with `asyncio.create_task()` and collect in a list
   - `await shutdown_event.wait()` to block until signal received
   - After signal: loop over tasks and call `task.cancel()`
   - `await asyncio.wait_for(asyncio.gather(*tasks, return_exceptions=True), timeout=5.0)` to ensure clean exit within 5 seconds
   - Pass `shutdown_event=shutdown_event` to `KalshiIngestor`, `PolymarketIngestor`, `_ttl_cleanup`
   - Add `shutdown_event` parameter to `_ttl_cleanup` and check it in the `while True` loop (break if set before/after sleep)

2. **albert/ingestor/base.py** — Add shutdown awareness:
   - Add `shutdown_event: asyncio.Event | None = None` parameter to `__init__`, store as `self._shutdown_event`
   - In `run()`, at the top of the `while True` loop, check `if self._shutdown_event.is_set(): return` before `await self._connect_and_stream()`

3. **albert/ingestor/kalshi.py** — Forward shutdown_event:
   - Add `shutdown_event: asyncio.Event | None = None` parameter to `__init__`
   - Pass it through: `super().__init__(bus, shutdown_event=shutdown_event)`

4. **albert/ingestor/polymarket.py** — Forward shutdown_event:
   - Same pattern as KalshiIngestor

5. **albert/execution/engine.py** — Check shutdown in both queue consumers:
   - In `handle_market_data()`: add `if self._shutdown_event.is_set(): return` BEFORE and AFTER `await self._market_data_queue.get()`
   - In `handle_orders()`: add `if self._shutdown_event.is_set(): return` AFTER `await self._order_queue.get()` (already checked before)

6. **albert/portfolio/tracker.py** — Check shutdown in both queue consumers:
   - In `handle_fills()`: add `if self._shutdown_event.is_set(): return` AFTER `await fills_queue.get()` (already checked before)
   - In `handle_market_data()`: add `if self._shutdown_event.is_set(): return` BEFORE and AFTER `await market_data_queue.get()`

7. **albert/strategies/engine.py** — Check shutdown after queue.get():
   - In `run()` after `event = await self._queue.get()`, add `if self._shutdown_event.is_set(): return`

**Why this works:** Cancelling a task that is blocked on `await queue.get()`, `await ws.recv()`, or `await asyncio.sleep()` raises `CancelledError` in the coroutine. The `asyncio.gather(return_exceptions=True)` collects these cleanly. The explicit `shutdown_event.is_set()` checks ensure tasks that just received an event during shutdown drop it and return immediately.
